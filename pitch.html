Turning never-ending CVE toil into a hands-free remediation loop - with Devin doing the work.

Presented by Deon Menezes  ·  live on a fork of Apache Superset
What · The problem

Dependency security is endless toil for engineering teams

  • Scanners file a steady drip of CVE issues - every week, forever
  • Each one is the same grind: triage → bump → fix conflicts → test → PR → merge
  • Low-value, high-interrupt work that pulls senior engineers off the roadmap
  • Skip it and security debt and audit risk compound
Important, tedious, and never actually done - the perfect job to automate.

The idea

An event-driven loop where Devin does the fixing

A CVE is detected → fixed → verified → merged, automatically3
How · The demo

The loop in action

  • A scan files a devin-remediate issue for each new CVE
  • Devin opens a scoped PR - “security: upgrade X → Y”, “Closes #N”
  • The engine posts the deps-verify check; branch protection requires it
  • Devin auto-reviews scope + green check, then squash-merges and the issue closes
  • A live control room shows every run, PR link, and ACU spent
One CVE issue in → one merged, verified pull request out. No human on the happy path.

How · Key decisions in the code
  • Group by package - one PR per package, not one per CVE
  • Guardrails: no published fix, or a major-version bump → held for a human
  • The engine is the CI - posts a first-class deps-verify status (private fork, Actions off)
  • Dry-run, a run ledger, and a report so a lead can see it working

Why · Why Devin, not a bot

Dependabot flags it. Devin closes it.

  • A bot bumps the pin, then leaves a red PR for a human
  • The real cost is the fix with consequences: API changes, lockfile regen, dependency conflicts, broken tests
  • Devin does the coding work a script can’t, then writes a PR a reviewer trusts
  • So route it: cheap tools clear trivial bumps, Devin takes the long tail that lands on an engineer’s desk
The bump is the easy 20%. Devin does the 80% that reaches a human today.

  • Multi-repo, multi-ecosystem - npm, cargo, go.mod via per-ecosystem parsers
